# Specforge V2 Context Packages

## Status

Accepted

## Purpose

This document defines the Context Package model used by Specforge.

Context Packages are responsible for transforming project knowledge into AI-consumable context.

They act as the bridge between the Knowledge Layer and AI systems.

The primary purpose of a Context Package is to provide an AI with the minimum amount of information required to make high-quality decisions while preserving important project understanding.

Context Packages are one of the most important outputs generated by Specforge.

---

# Introduction

Most AI workflows operate directly on prompts.

Prompts are temporary.

Knowledge is durable.

The challenge is transforming durable knowledge into reusable AI context.

Context Packages solve this problem.

---

# Definition

## Context Package

A Context Package is a curated collection of knowledge artifacts assembled for a specific AI objective.

A Context Package answers:

"What does the AI need to know before performing this task?"

A Context Package is not a prompt.
A Context Package is not a specification.

It is structured project understanding prepared for AI consumption.

---

Knowledge

# Responsibilities

Context Packages exist to:

- reduce context reconstruction
- improve AI consistency
- preserve project understanding
- prevent important omissions
- reduce contradictory outputs
- improve traceability

---

# Context Package Structure

Every Context Package should contain:

## Package Metadata

- Package ID
- Name
- Version
- Creation Date
- Generated By
- Purpose

## Objective

What is the AI trying to accomplish?

Examples:

- Generate requirements
- Generate architecture
- Review code
- Create tests
- Create harnesses

## Relevant Knowledge

May include:

- Product DNA
- Discoveries
- Decisions
- Constraints
- Principles
- Risks
- Assumptions

Only relevant knowledge should be included.

## Required Context

Information that must be understood before generation.

## Optional Context

Useful information that may improve outputs.

## Excluded Context

Information intentionally omitted.

## Source References

Links to original knowledge artifacts.

Examples:

- DEC-001
- DISC-014
- CONST-002

---

# Context Package Categories

## Project Context Package

Provides understanding of the entire project.

## Feature Context Package

Provides understanding of a specific feature.

## Architecture Context Package

Provides architectural understanding.

## Testing Context Package

Provides testing context.

## Harness Context Package

Provides guidance for AI agents.

---

# Context Selection Principles

## Relevance

Only include information related to the objective.

## Traceability

Sources must remain discoverable.

## Consistency

Avoid conflicting information.

## Completeness

Include everything required.

## Efficiency

Avoid unnecessary tokens.

# Context Compression

Compression should remove:

- duplication
- irrelevant knowledge
- obsolete information

Compression should never remove:

- rationale
- constraints
- critical decisions
- critical discoveries

---

# Context Quality Evaluation

A Context Package is high quality when:

- relevant information is present
- irrelevant information is absent
- sources remain traceable
- contradictions are minimized
- AI outputs remain consistent

# AI Responsibilities

AI systems using Specforge should:

- request relevant Context Packages
- respect package boundaries
- preserve traceability
- identify missing context
- recommend package improvements

AI should not silently ignore Context Package information.

---

# Context Package Anti-Patterns

## Everything Package

Including the entire project regardless of relevance.

## Opaque Package

Removing traceability.

## Knowledge Compression

Removing rationale to save tokens.

## Stale Package

Using outdated knowledge.

## Context Fragmentation

Creating multiple contradictory packages.
